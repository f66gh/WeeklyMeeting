## 师兄给的方向

**并行处理云化**

基于端云一体化架构，将计算服务容器化部署到容器集群，通过任务分片和分布式队列等形式实现并行处理。建立智能调度机制，根据负载情况动态分配云端资源，通过水平扩展和资源池化技术实现云版处理能力达到单机版3倍以上。

任务：**科学计算和物理仿真**

具备以下任务特性

* 任务易于拆分   
* 任务同质化+数据异质化        
* 计算密集型       
* 任务依赖关系明确       
* 延迟不敏感        
* 吞吐量优先        
* 允许排队       
* 有状态/中间结果重要       
* 高精度要求       
* 高频率数据交换       
*  资源需求可预测        
* 硬件加速支持

**场景总结**

延迟非敏感的计算密集型任务，不会和边缘计算有关；最终通过【专用】集群承载 类似的其它任务：

* XR场景/模型预渲染任务
* AI训练
* 延迟非敏感的推理（大模型）

**可选决策内容**

Case 1（负载均衡+资源调度）: 多种通用的Worker微服务承担异构任务（实际上直接简化成Serverless可能更靠谱），于是Worker间会有的中间数据传输问题 

Case 2（服务放置+资源调度）：将大任务按照任务分配，设置若干专用的Worker，按照{系统剩余资源，预期资源}等手段，动态对Worker进行调度

**考虑的因素**（目标/约束）

* 专用集群的运营成本 
* 吞吐量
* 等待时间

### 节能类

云端并行化不完善，以及受到移动端通信问题的影响，开发出==**节能**==的资源管理算法

* Energy-Efficient Real-Time Job Mapping and Resource Management in Mobile-Edge Computing **移动边缘计算中的节能实时任务映射和资源管理**

* 移动边缘计算（MEC）已成为一种使物联网（IoT）设备能够处理计算密集型任务的理想范式。由于服务器上任务处理算法的并行化不完善以及物联网设备移动性对无线网络中数据通信质量的影响，在任务调度过程中联合考虑服务器资源分配和物联网设备移动性对于充分利用 MEC 至关重要，而这一点在现有研究中往往被忽视。通过联合考虑任务调度、服务器资源分配和物联网设备移动性，我们研究了具有通信和计算争用的 MEC 中带有时限约束的任务卸载和资源管理问题，旨在最大化物联网设备的总节能效果。对于问题离线版本，其中任务信息是预先已知的，我们将其建模为整数线性规划问题，并提出了一种具有常数性能保证的近似算法 LHJS。对于在线版本，其中任务信息仅在发布时已知，我们提出了一种启发式算法 LBS，该算法在每次任务发布时被调用。 最后，我们使用来自实际应用的参数进行实验，以评估其性能。


### 安全类

在云端协同实现节能的前提下，必须保护边缘网络用户的精确位置信息（好6的角度）

* GEES: Enabling Location Privacy-Preserving Energy Saving in Multi-Access Edge Computing  **GEES：在多接入边缘计算中实现位置隐私保护节能**
* 全球 5G 网络的部署导致边缘服务器数量大幅增加，以支持边缘网络用户日益增长的低服务延迟需求。然而，全天候运行边缘服务器会导致巨大的能源消耗和过度的碳排放。在新多接入边缘计算（MEC）架构中，实现节能的边缘资源提供是达成可持续发展目标的关键。近期，已有多种方法被提出以解决云计算和 MEC 中的节能需求响应问题。然而，必须始终提供边缘网络用户的精确位置信息，这会牺牲用户隐私。为在 MEC 中节能的同时保护边缘网络用户的地理位置隐私，我们系统地提出了这一位置隐私保护边缘需求响应（LEDR）问题。为高效且有效地解决 LEDR 问题，我们通过结合差分地理模糊化技术，在保护用户隐私的同时，通过理论分析进行推断，以最大化系统效用和能源效率，提出了名为 GEES 的系统。 基于综合真实世界数据集进行了广泛而全面的实验，结果表明，在能效、用户隐私和系统效用方面，GEES 平均比代表性方法高出 23.02%、31.47%和 17.29%。

在保证微服务架构的安全同时提升了工作流的性能，设计了一种针对基于微服务的应用程序的安全调度模型。

* **基于 D3QN 的云环境中微服务工作流安全调度** D3QN-based secure scheduling of microservice workflows in cloud environments

* 近年来，由于其松散耦合的特性，微服务架构被广泛应用于软件设计，包括云中的科学工作流调度。随着容器技术的发展，其快速启动和低开销使其成为微服务的首选部署技术。然而，当前微服务架构的一个关键问题是暴露私有和敏感数据的风险。现有工作研究了并探索了微服务的新架构，但隐私和时间之间的权衡仍未得到充分考虑。为了填补这一空白，我们首先设计了一个考虑用户和云提供者安全级别的安全模型，并提出了一种满足预算约束和任务隐私要求的工怍流调度算法。我们将安全调度问题转化为马尔可夫决策过程（MDP），并提出了基于双斗篷深度 Q 网络（D3QN）的微服务工作流调度算法（D3MWS），该算法不仅能在预算约束下最小化完成时间，还能满足工作流任务的安全要求。 最后，选择了云计算中著名的 workflow 应用程序进行实验。实验结果表明，所提出的算法在满足微服务的安全要求的同时，与其他算法相比，在不同任务数量下将每个 workflow 的平均任务完成时间减少了 10.91%。


引入针对工业边缘计算的安全信任模型。

* 工业网络物理系统中的可信微服务编排用于安全边缘计算 **Trusted Microservice Orchestration for Secure Edge Computing in Industrial Cyber-Physical Systems**
* 工业物联网（IIoT）和计算技术的快速渗透显著改变了工业物理系统（ICPS）的格局。云雾自动化等最近出现的颠覆性范式提倡在云/雾层开放部署 ICPS。这些范式旨在通过利用云/雾技术的计算能力来增强系统模块化和效率。ICPS 产生的大量多样化数据很容易因带宽和延迟挑战而使传统云解决方案不堪重负。基于边缘的分析通过在数据源附近处理数据提供了一种有前景的解决方案，然而在边缘部署高级分析由于边缘设备的计算、内存和能源限制而具有挑战性。最近，微服务已成为在 ICPS 中提供轻量级和模块化分析服务的理想候选。然而，微服务引入了严重的安全漏洞和运营复杂性，特别是在异构和动态的工业环境中。 现有的微服务编排平台是为稳定的云环境设计的，因此无法满足工业边缘计算独特的安全和适应性要求。为此，我们提出了一种针对工业控制物理系统（ICPS）中安全边缘计算的新型漏洞感知微服务编排框架。我们首先构想了能够检测微服务行为异常的信任模型，然后使用最先进的平台进行广泛的模拟，以验证我们的信任模型在增强去中心化工业边缘计算环境中的网络物理安全性和系统韧性方面的有效性。

本文探讨了区块链与云端协同下的加密和共享数据问题，并提出了一种框架。

* A data encryption and file sharing framework among microservices-based edge nodes with blockchain **一个基于区块链的微服务边缘节点之间的数据加密和文件共享框架**
* 云计算向 X-as-a-Service 模式的发展以及微服务架构的采用，显著推动了该领域的成熟。随着这些技术的进步，它们带来了数据一致性、交易可追溯性和高效信息处理等挑战，特别是在边缘计算环境中。本文探讨了将微服务架构、边缘计算和区块链技术相结合，以应对这些挑战，重点关注安全文件共享。我们提出了一种框架，该框架使用身份认证并利用以太坊区块链、IPFS 和 HDFS 实现去中心化、防篡改的文件共享。我们在 FsMbe 系统中的实现，在通过以太坊进行身份认证后，使用分布式网络的 DHT 获取数据，并采用多轮 AES 加密进行数据共享。我们进行了广泛的实验，以评估该框架在不同场景下的性能，包括边缘组内微服务之间的小数据通信以及较大组内边缘节点之间的大文件传输。 该框架展示了稳定的数据传输性能，尤其是在小数据负载的情况下，即使客户端数量增加也是如此。这项研究为边缘计算和微服务环境中的安全高效数据管理做出了贡献，为去中心化文件共享提供了一种性能和安全性增强的可靠解决方案。

### 提升资源调度（扩展）性能

现有微服务架构自动扩展器不适用于资源受限的环境，并且Pod的固有延迟加剧问题。所以提出了一种自动扩展器促进微服务之间的资源交换，并提出了Pod启动和终止带来的延迟。

* Towards resource-efficient reactive and proactive auto-scaling for microservice architectures **面向微服务架构的资源高效反应式和主动式自动伸缩**
* 微服务架构在学术界和工业界都日益普及，为软件开发和部署提供了更高的敏捷性、弹性和可维护性。为了简化微服务架构的扩展操作，像 Kubernetes 这样的容器编排平台配备了水平 Pod 自动扩展器（HPAs），用于调整微服务的资源以适应波动的负载。然而，现有的 HPAs 不适用于资源受限的环境，因为它们基于微服务的单个资源容量做出扩展决策，导致服务不可用、资源管理不当和财务损失。此外，初始化和终止微服务 Pod 的固有延迟阻碍了 HPAs 及时响应负载波动，进一步加剧了这些问题。为了解决这些问题，我们分别提出了 Smart HPA 和 ProSmart HPA，即反应式和主动式的资源高效水平 Pod 自动扩展器。Smart HPA 采用反应式扩展策略，促进微服务之间的资源交换，优化资源受限环境下的自动扩展。 对于 ProSmart HPA，我们开发了一种基于机器学习的资源高效扩展策略，该策略主动管理资源需求，以解决微服务 Pod 启动和终止引起的延迟，同时在资源受限的环境中实现预占式资源共享。我们的实验结果表明，Smart HPA 的表现优于 Kubernetes 基准 HPA，而 ProSmart HPA 通过减少资源过度使用、过度配置和配置不足，并将更多资源分配给微服务应用，超越了 Smart HPA 和 Kubernetes HPA。

==扩展微服务仍然面临请求意外峰值或突发性挑战，并且降低性能。为了改善这个问题提出了一种框架，为差异化资源调度提供合适的弹性扩展策略==

* StatuScale: Status-aware and Elastic Scaling Strategy for Microservice Applications **StatuScale：微服务应用程序的状态感知和弹性扩展策略**

* 微服务架构已将传统的单体式应用程序转变为轻量级组件。扩展这些轻量级微服务比扩展服务器更有效。但是，扩展微服务仍然面临请求意外峰值或突发性挑战，这些挑战难以检测，并且可能会立即降低性能。为了应对这一挑战并确保基于微服务的应用程序的性能，我们提出了一种名为 StatuScale 的状态感知弹性扩展框架，该框架基于负载状态检测器，可以为垂直扩展中的差异化资源调度选择合适的弹性扩展策略。此外，StatuScale 还采用了水平扩展控制器，该控制器利用全面评估和资源减少来管理每个微服务的副本数量。我们还提出了一个名为 correlation factor 的新指标来评估资源使用效率。最后，我们使用 Kubernetes（一个开源容器编排和管理平台）以及来自阿里巴巴的真实跟踪来验证我们的方法。实验结果表明，与基线相比，所提出的框架可以将 Sock-Shop 应用程序的平均响应时间减少 8.59% 到 12.34%，将 Hotel-Reservation 应用程序的平均响应时间减少 7.30% 到 11.97%，减少服务级别目标违规，并提供更好的资源使用性能。

==现有扩展微服务的方法忽略了关键微服务与非关键微服务资源间的资源可替代性。==

* Grad: Intelligent Microservice Scaling by Harnessing Resource Fungibility **Grad：通过利用资源可替代性实现智能微服务扩展**

* 微服务应用程序通常与其他服务一起部署，以提高资源利用率。然而，这种做法也会导致显著的资源竞争。虽然现有研究主要关注扩展导致性能下降的关键微服务，以减轻高度干扰环境下端到端延迟违反服务水平协议的情况，但它们往往忽视了扩展非关键微服务以优化资源效率的潜在优势。在本文中，我们介绍了 Grad，一个通过利用关键微服务和非关键微服务之间的资源可替代性来实现智能微服务扩展的框架。针对资源可替代性在扩展过程中的动态特性带来的挑战，Grad 包含了三个关键组件。首先，Grad 采用模块化学习方法来分析单个微服务在不同环境条件下的延迟。利用从该分析中提取的梯度信息，Grad 设计了一个可扩展的优化模块，以动态选择最佳的微服务扩展集。 为了快速缓解 SLA 违规，Grad 还部署了一个精确的端到端延迟预测器，作为模拟器以获取实时反馈。我们在集群中使用真实的微服务基准测试和生产跟踪来评估 Grad，证明其与最先进解决方案相比能够减少 49.1% 的资源使用率，并将 SLA 违规的概率降低 3.7× 。

我们提出了一种基于服务水平协议（SLA）的节能调度算法，以减少部署在 Kubernetes 中的微服务的能耗。

* Energy-aware Scheduling Algorithm for Microservices in Kubernetes Clouds Kubernetes **云环境中微服务的节能调度算法**
* 越来越多的应用程序以微服务网格的形式组织，这些微服务可以部署在流行的容器编排平台 Kubernetes 上。Kubernetes 为复杂的微服务网格应用程序提供自动化管理、高可用性、弹性扩展和跨云兼容性。尽管 Kubernetes 是管理容器的强大工具，但其默认的调度算法和现有的容器调度研究主要针对单体应用程序设计。它们未能考虑不同微服务的资源消耗差异，以及这些微服务的心跳机制导致的 CPU 消耗，从而造成能源浪费和效率低下。因此，我们提出了一种基于服务水平协议（SLA）的节能调度算法，以减少部署在 Kubernetes 中的微服务的能耗。该算法通过将整体 Pod 的通信频率除以 Pod 之间的网络流量，并根据 Pod 中运行的微服务的资源消耗来优先考虑 Pod 的资源消耗。 此外，设计并应用了一种改进的麻雀搜索算法（ISSA），根据 Pod 的通信频率和资源消耗优先级来打包 Pod，以实现确保 SLA 和减少能耗的目标。实验结果表明，与最新的容器调度算法相比，云环境中的 Kubernetes 集群能耗至少降低了 5%。

### 故障检测

用广泛的分类法涵盖工业中大规模微服务项目中常见的故障。

* A Taxonomy of Integration-Relevant Faults for Microservice Testing **面向微服务测试的集成相关故障分类法**

* 微服务已成为一种流行的架构范式，为软件开发提供了一种灵活且可扩展的方法。然而，它们的分布式特性和多样化的技术栈引入了固有的复杂性，超越了单体系统的复杂性。微服务的集成面临着诸多挑战，从通信故障到兼容性问题，都会影响系统的可靠性。了解这些分布式组件中的故障对于防止缺陷、制定测试策略和实施鲁棒性测试至关重要。尽管这些软件系统具有重要意义，但现有的分类法有限，因为它们主要关注非功能性属性或缺乏实证验证。为了解决这些差距，本文提出了一个广泛的分类法，涵盖了工业中大规模微服务系统中最常见的集成相关故障。通过系统文献综述和与行业专家进行的十次半结构化访谈，我们识别了真实世界微服务项目中常见的集成相关故障。 我们的最终分类法通过一项调查得到验证，该调查邀请了另外16位从业者参与，确认几乎所有故障类别（21/23）都至少被50%的受访者遇到过。


现有的微服务传统故障定位方法缺乏灵活性，LLM方法成本过高。为此我们提出了ThinkFL赋予轻量级LLM推理，提升效益和适应性。

* ThinkFL: Self-Refining Failure Localization for Microservice Systems via Reinforcement Fine-Tuning **ThinkFL: 通过强化微调实现微服务系统的自我完善故障定位**

* 随着现代微服务系统日益普及和复杂化——通常由数百甚至数千个细粒度、相互依赖的组件构成——它们更容易频繁出现细微故障。因此，确保系统可靠性依赖于准确高效的故障定位。基于小型模型的传统故障定位方法缺乏适应多样化故障场景的灵活性，而最新的基于 LLM 的方法存在两大主要限制：它们通常依赖僵化的调用工作流程，限制了模型动态探索最优定位路径的能力，并且需要资源密集型的推理，导致在实际部署中成本过高。为应对这些挑战，我们探索使用强化微调来赋予轻量级 LLM 推理和自我完善的能力，显著提升基于 LLM 的故障定位的成本效益和适应性。我们首先进行实证研究，以识别出三个对准确定位至关重要的关键能力。 基于这些见解，我们提出了一种渐进式多阶段 GRPO 微调框架，该框架集成了多因素故障定位评分器和思维递归行动器模块。由此产生的模型 ThinkFL 不仅在定位精度上优于现有的最先进 LLMs 和基线方法，而且将端到端定位延迟从分钟缩短到秒，展示了其在实际应用中的巨大潜力。

我们提出了一种混合事件向量序列HVS，能够更高效地识别服务与服务之间的耦合异常。

* **TraLogAnomaly: 基于混合事件序列的微服务系统异常检测方法** TraLogAnomaly: A microservice system anomaly detection approach based on hybrid event sequences

基于对比多模态表示聚类的无监督微服务系统异常检测方法（MAD-CMC）。
